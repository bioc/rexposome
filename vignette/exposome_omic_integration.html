Exposome Data Integration with Omic Data

Carles Hernandez-Ferer and Juan R. Gonzalez

9 January 2017

Abstract

This is an introductory guide to integration analysis between exposome and omics data with R package rexposome. The document ilustrates two types of analysis: 1) Association analysis, that are performed between exposome and a single omic data-set; and 2) Integration analysis where multiple data-sets, including exposome data, are analysed at the same time.

Package version: rexposome 0.99.43

Under this sections two types of analysis are explained: Association Analysis and Integration Analysis.

Both types of analysis requests a MultiDataSet object. While Association Analysis are performed using a MultiDataSet object that contains an ExposomeSet plus a SnpSet, MethylationSet or ExpressionSet, Integration Analysis needs a MultiDataSet object with, at last, two of these objects.

2.1 Associaction Analysis

library(rexposome)
library(MultiDataSet)

2.1.1 Exposome - Genome Data Associaction

The first step of an association analysis performed with rexposome is to create the MultiDataSet with the data to be analyzed.

The goal of this analysis is to perform a basic GWAS using the phenotypes stored in an ExposomeSet. Hence, a MultiDataSet with an ExposomeSet and a SnpSet is needed.

data("exposome", package = "rexposome")
ldBRGE("snp")
sampleNames(snp4) <- gsub("id", "SIMSAB", sampleNames(snp4))

Once the data is loaded a MultiDataSet object is created containing both data-sets:

mds <- createMultiDataSet()
mds <- add_snps(mds, snp4)
mds <- add_exp(mds, expo)

After creating the MultiDataSet created the common samples between sets are kept and the others discarded:

mds <- commonSamples(mds)
mds

##  . assayData: 2 elements
##     . elements: snps, exposures 
##  . featureData:
##     . snps: 817131 rows, 6 cols (chromosome, ..., allele.1)
##     . exposures: 104 rows, 8 cols (Family, ..., LOD)
##  . rowRanges:
##     . snps: YES
##     . exposures: NO
##  . phenoData:
##     . snps: 145 samples, 7 cols (pedigree, ..., affected)
##     . exposures: 145 samples, 5 cols (asthma, ..., age)

The association tests is done using the method assocSNP. The arguments of this method are:

args(assocSNP)
## function (object, design, family = "binomial", filter.callrate = 0.98, 
##     filter.maf = 0.05, filter.pvalHWE = 0.001, ...) 
## NULL

The object argument is filled with the MultiDataSet object created previously. The design is a formula where the dependent variable are the genotypes and the dependent the heath outcome. family argument must indicate the type of the heath outcome.

The following are the health outcomes in the ExposomeSet:

phenotypeNames(expo)
## [1] "asthma" "BMI"    "sex"    "age"

A GWAS for asthma using assocSNP is done as:

gwas <- assocSNP(mds, asthma~sex+age)

The result is an object of class ResultSet:

##  . #results: 1 
##  . featureData:  2 
##     . exposures: 104x8
##     . snps: 817131x6

This object allows to plot the results and draw a QQ plot and a Manhattan plot:

plotAssociation(gwas, type="qq")

plotAssociation(gwas, type="manhattan")

sampleNames(mset4) <- gsub("id", "SIMSAB", sampleNames(mset4))

mds <- createMultiDataSet()

After obtaining the common samples, the association analysis between CpGs and exposures is done using assocME. This method requires a MultiDataSet object and a formula with the covariates. The optional argument select allows to perform the association on a sub-set of exposures (if set = "exposures) or sub-set of outcomes(if set = "phenotypes").

ewas <- assocME(mds, ~sex+age, 
    select=c("logpb", "lcrCo_T1", "lTHM_T1", "Ben_preg", "garden_pesticides"))

2.2 Integration Analysis

The aim on the integration analysis is to analyze multiple data-sets in a single shot, looking for similar behaviors in terms of features and through the data-sets. rexposome allows to perform this analysis using two strategies: multiple co-inertia analysis (mcia) and sparse multiple canonical correlation analysis (mcca).

To perform an integration analysis using exposome, transcriptome and proteome data a MultiDataSet object is required with the thee data-sets.

data("exposome", package = "rexposome")
ldBRGE("gexp")
## ldBRGE("prot")
sampleNames(gexp4) <- gsub("id", "SIMSAB", sampleNames(gexp4))

All the data-sets needs to be complete. So the exposures should be imputed:

expo <- impute(expo)
mds <- createMultiDataSet()
mds <- add_prot(mds, prot4)
mds <- add_genexp(mds, gexp4)
mds <- add_exp(mds, expo)

Once the data-sets is created, we look for the common samples.

mds <- commonSamples(mds)
mds
## Object of class 'MultiDataSet'
##  . assayData: 3 elements
##     . elements: protein, ..., expression 
##  . featureData:
##     . protein: 47 rows, 2 cols (A, B)
##     . expression: 67528 rows, 11 cols (transcript_cluster_id, ..., gene_assignment)
##     . exposures: 104 rows, 8 cols (Family, ..., LOD)
##  . rowRanges:
##     . protein: NO
##     . expression: YES
##     . exposures: NO
##  . phenoData:
##     . protein: 37 samples, 3 cols (A, ..., B)
##     . expression: 37 samples, 76 cols (Sample_Name, ..., Project)
##     . exposures: 37 samples, 5 cols (asthma, ..., age)

Both integration analysis are done using the method crossomics. The argument method can takes the values "mcia" and "mcca".

crs1 <- crossomics(mds, method="mcia")
crs2 <- crossomics(mds, method="mcca")
## 12345678910111213141512345678

The method crossomics returns a ResultSet that can be used in the method plotIntegration to plot the results.
